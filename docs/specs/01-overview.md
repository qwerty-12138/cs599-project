# 01 — 项目总览与需求规格

## 1.1 项目目标

构建一个具备以下能力的 AI Agent 对话系统：

1. **有记忆的 Agent**：短期记忆（Redis）+ 会话持久化（JSON文件），对话上下文自动保存
2. **MCP 协议工具调用**：可配置 MCP 服务器，对话中可选择启用/禁用特定工具
3. **条件 RAG 检索**：仅在问题与知识库相关时触发 RAG，否则直接使用 LLM 回答
4. **Docker 全栈部署**：所有基础设施容器化，一键启动
5. **流式输出**：SSE 实时推送 token，工具调用等阻塞场景显示进度动画
6. **对话取消**：用户可随时终止当前生成，该轮对话不进入持久化
7. **完善前端界面**：参照参考图设计，美观现代
8. **评估体系**：RAGAS 基准测试 + LLMOps 可观测性

## 1.2 用户故事

## 1.3 功能需求清单

### FR-1: 对话系统
- FR-1.1: 支持多轮对话，保持上下文
- FR-1.2: 支持流式输出 (SSE)
- FR-1.3: 支持取消正在进行的生成
- FR-1.4: 取消的对话不进入持久化记忆
- FR-1.5: 显示工具调用进度动画

### FR-2: 意图分类与路由
- FR-2.1: 自动识别用户意图 (RAG 相关 / 闲聊 / 工具调用)
- FR-2.2: RAG 相关问题：先检索知识库，再生成回答
- FR-2.3: 非 RAG 问题：直接使用 LLM 回答，不携带检索上下文

### FR-3: 记忆系统
- FR-3.1: 短期记忆存储在 Redis，24 小时 TTL
- FR-3.2: 会话记录持久化存储在 JSON 文件
- FR-3.3: 每次消息后自动保存到 JSON 文件
- FR-3.4: 容器重启后自动加载会话记录

### FR-4: MCP 集成
- FR-4.1: 支持配置多个 MCP 服务器
- FR-4.2: 前端面板可启用/禁用服务器和具体工具
- FR-4.3: 工具调用走 ReAct 循环
- FR-4.4: 服务器启用时自动加载工具到可用列表
- FR-4.5: 服务器禁用时自动从可用列表移除工具
- FR-4.6: MCP 工具列表持久化存储，容器重启后自动恢复

### FR-5: RAG 管道
- FR-5.1: 支持文档上传 (PDF, TXT, Markdown)
- FR-5.2: 文档分块 + 嵌入 + 存储
- FR-5.3: 混合检索 (向量 + BM25)
- FR-5.4: 条件路由：仅相关时使用

### FR-6: Docker 部署
- FR-6.1: 所有服务 Docker 化
- FR-6.2: Docker Compose 一键编排
- FR-6.3: 沙箱隔离工具执行

### FR-8: 评估与可观测性
- FR-8.1: RAGAS 评估指标
- FR-8.2: 响应延迟监控
- FR-8.3: 工具调用成功率统计
- FR-8.4: Grafana 可视化仪表盘

## 1.4 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | P95 延迟 < 5s (含 RAG 检索) |
| 可用性 | Docker 部署后所有服务健康检查通过 |
| 安全 | API Key 环境变量注入，输入 XSS 防护 |
| 可维护性 | 模块化设计，清晰目录结构 |
| 兼容性 | 前端支持 Chrome 120+, Edge 120+ |
